Tutorial de GitHub

Aprende paso a paso cómo inicializar un repositorio, hacer merge y subir los cambios a GitHub

1. Inicializar el repositorio

El primer paso es inicializar un nuevo repositorio en Git. Para ello, utiliza el siguiente comando en la terminal dentro de tu carpeta de proyecto:

git init

Este comando crea una nueva carpeta oculta .git en tu proyecto, donde Git almacenará la información necesaria para el control de versiones.

2. Agregar archivos al repositorio

Una vez inicializado, agrega los archivos que quieres que Git rastree:

git add .

Esto añadirá todos los archivos del proyecto al área de preparación (staging area). Si solo deseas agregar un archivo en específico, usa git add nombre_archivo.

3. Hacer commit

Después de agregar los archivos, el siguiente paso es crear un commit, que representa una "foto" del estado de tu proyecto en ese momento:

git commit -m "Mensaje descriptivo del commit"

Asegúrate de escribir un mensaje claro para identificar qué cambios hiciste en este commit.

4. Subir a GitHub (Push)

Primero, debes vincular tu repositorio local con uno remoto en GitHub. Para ello, usa el siguiente comando:

git remote add origin https://github.com/tu-usuario/nombre-repositorio.git

Una vez vinculado, puedes subir los cambios al repositorio remoto con:

git push -u origin master

Este comando subirá los commits al branch principal (master o main) de tu repositorio en GitHub.

5. Crear una nueva rama (branch)

Para trabajar en nuevas funcionalidades sin afectar el código principal, crea una nueva rama con:

git checkout -b nueva-rama

Esto crea y cambia a la nueva rama. Aquí puedes hacer cambios y commits sin modificar la rama principal.

6. Hacer merge de las ramas

Cuando hayas terminado de trabajar en tu rama y estés listo para incorporar esos cambios en la rama principal, cambia a la rama principal:

git checkout master

Luego, haz el merge de tu rama con la principal:

git merge nueva-rama

Git unirá los cambios de la nueva rama con la principal. Si hay conflictos, Git te pedirá que los resuelvas manualmente.

7. Subir cambios después del merge

Finalmente, una vez hecho el merge, sube los cambios nuevamente a GitHub con:

git push origin master

¡Y listo! Tus cambios estarán en el repositorio remoto y habrás finalizado el proceso de trabajo con Git y GitHub.
